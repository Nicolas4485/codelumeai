## Notes on this file's language (Python)

This file uses these Python building blocks. Skim once, then refer back as needed.

- **`class Name:`** — a template / blueprint. Defines a *kind of thing*. Each one created from it is called an *instance*. Cookie-cutter and cookies.
- **`def name(...)`** — a function definition. When defined inside a `class`, it's called a *method* — and the first argument `self` refers to the specific instance the method is called on.
- **`__init__`** — the *setup method* that runs automatically when a new instance is created. The double underscores mark it as something Python calls for you, not something you call directly.
- **Decorator (`@something`)** — a label written above a class or function that adds extra behavior. `@dataclass` says "this class is a data holder; Python, auto-generate the routine setup code." `@property` says "this method is accessed like a field, without parentheses."
- **Underscore prefix (`_name`)** — a Python convention meaning "internal — not part of the public interface." A naming hint, not enforced by the language.
- **Type hints** — `x: str` means "x holds a string." `-> list[str]` means "returns a list of strings." `str | None` means "either a string or nothing." Type hints are *labels*, not enforced rules — Python doesn't check them at runtime.
- **`field(default_factory=list)`** — gives every new instance its *own fresh* empty list. (Writing `= []` instead would secretly share one list across all instances — a famous Python footgun.)
- **`**kwargs`** — collects any number of keyword arguments into a dict. `def matches(self, **kwargs)` lets the caller pass whatever named arguments they want.
- **List/set comprehension** — Python's compact way to build a list or set: `[t for t in things if t.ok]` reads as "for each `t` in `things`, keep it if `t.ok` is true."
